Второй, более лёгкий путь установки — для тех, кто уже разговаривает с
Claude Code: делает то же самое, что `install.sh` для клиента Claude Code, но
проводя те же шаги прямо из этого чата, вместо того чтобы просить пользователя
вставить строку `curl | bash` в терминал. Это надстройка над `install.sh`, а
не замена — `install.sh` всё ещё остаётся способом зарегистрировать
Codex/ChatGPT Desktop/Claude Desktop, и у него есть сценарии обслуживания
`--relogin`/`--uninstall`, которые этот скилл не пытается воспроизводить.
Если пользователь просит именно это — направьте его к `install.sh` (см.
README проекта).

## Почему нельзя просто вызвать `install.sh`

`install.sh` читает `api_id`/`api_hash` и проводит вход в Telegram по QR через
`/dev/tty`, которого у вызова инструмента `Bash` из Claude Code нет. Этот
скилл не пытается подделать TTY, а вовсе обходит шаг входа: он пишет
`config.env` с учётными данными, но **без строки сессии** — это переводит
сервер при следующем запуске во встроенный режим «только вход». В этом режиме
он открывает ровно два инструмента, `telegram_login_start` и
`telegram_login_check`, которые проводят QR-вход прямо через чат (тот же
механизм, что использует бандл `.mcpb` для Claude Desktop — см. `login.py` и
шаг 6б в `PLAN.md`). Разделение такое: этот скилл делает всё вплоть до
регистрации сервера, а *следующая* сессия чата завершает вход, буквально
вызывая эти два инструмента.

## Шаги

1. **Проверить, не настроено ли уже.** Если инструменты Telegram (например,
   `list_chats`, `get_history` или `telegram_login_start`) уже видны в этой
   сессии — ставить нечего: сказать об этом пользователю и, если видны
   только инструменты входа, сразу перейти к шагу 7.

2. **Проверить предпосылки** через `Bash` (TTY здесь нигде не нужен):
   - `command -v python3.13 python3.12 python3.11 python3.10 python3` —
     годится первый найденный с версией ≥ 3.10.
   - Если ни один не подошёл, проверить `command -v uv`. Если нет и его —
     спросить пользователя (в чате, не в терминале), ставить ли `uv`
     командой `curl -LsSf https://astral.sh/uv/install.sh | sh` (без sudo,
     ставится в `~/.local/bin`). Если отказался — направить на
     https://www.python.org/downloads/ и остановиться здесь.

3. **Создать venv и поставить пакет**, повторяя `setup_python_and_venv`/
   `install_package` из `install.sh`. Версия берётся из последнего релиза на
   GitHub, с откатом на `main`, если релиза ещё нет или API недоступно — та
   же логика, что и в `install.sh`:
   ```bash
   mkdir -p ~/telegram-mcp-ag
   <python> -m venv ~/telegram-mcp-ag/.venv   # или: uv venv --seed --python 3.12 ~/telegram-mcp-ag/.venv
   ~/telegram-mcp-ag/.venv/bin/python -m pip install --upgrade pip -q

   ref="$(curl -fsSL https://api.github.com/repos/agroznykh/telegram-mcp-ag/releases/latest 2>/dev/null \
       | grep -o '"tag_name": *"[^"]*"' | head -1 | sed 's/.*"\([^"]*\)"$/\1/')"
   ref="${ref:-main}"
   ~/telegram-mcp-ag/.venv/bin/python -m pip install -q "git+https://github.com/agroznykh/telegram-mcp-ag.git@${ref}"
   ```

4. **Спросить учётные данные Telegram API в чате.** Сказать пользователю
   открыть https://my.telegram.org/apps (войти под своим номером телефона,
   открыть «API development tools») и дать вам `api_id` и `api_hash`.
   Проверить перед записью: `api_id` должен состоять только из цифр;
   `api_hash` обычно 32 шестнадцатеричных символа — если это не так,
   уточнить у пользователя, использовать ли как есть, а не отклонять сразу
   (у некоторых аккаунтов он отображается иначе).

5. **Записать `config.env` без строки сессии**, используя штатный писатель
   проекта, чтобы права доступа и экранирование совпадали с тем, что ожидает
   сервер:
   ```bash
   ~/telegram-mcp-ag/.venv/bin/python -c '
   from telegram_mcp_ag import config
   config.write_values({
       "TELEGRAM_API_ID": "<api_id>",
       "TELEGRAM_API_HASH": "<api_hash>",
       "TELEGRAM_EXPOSED_TOOLS": "read-only",
       "TELEGRAM_DEVICE_MODEL": "telegram-mcp-ag (claude-code-setup)",
   })'
   ```
   Это создаёт `~/telegram-mcp-ag/config.env` с правами `600`. `
   TELEGRAM_SESSION_STRING` намеренно не указывается — именно его отсутствие
   переводит сервер в режим «только вход» при следующем запуске.

6. **Зарегистрировать сервер в Claude Code:**
   ```bash
   claude mcp remove -s user telegram-mcp-ag 2>/dev/null || true
   claude mcp add -s user telegram-mcp-ag -- ~/telegram-mcp-ag/.venv/bin/telegram-mcp-ag
   ```

7. **Передать эстафету новой сессии.** Claude Code загружает MCP-серверы при
   запуске, поэтому текущий чат не может подхватить только что
   зарегистрированный сервер сам. Сказать пользователю прямо: начните новую
   сессию Claude Code (новое окно/вкладка, либо выйти и перезапустить
   `claude`), и в ней сказать что-то вроде «подключи мой аккаунт Telegram».
   Сервер поднимется в режиме «только вход» и откроет
   `telegram_login_start`/`telegram_login_check`; вызов
   `telegram_login_start` покажет QR-код прямо в чате (а также откроет его
   локальным файлом-картинкой и даст ссылку `tg://login` как запасной
   вариант) — отсканировать через Telegram → Настройки → Устройства →
   Подключить устройство, а `telegram_login_check` завершит вход после
   сканирования. После этого инструменты чтения и транскрипции станут
   доступны в той же сессии — без дополнительного перезапуска.

8. **Если у аккаунта есть облачный пароль 2FA**, `telegram_login_check`
   автоматически попробует показать системное окно ввода пароля ОС. На
   headless-машине, где такое окно показать нельзя, он сообщит об этом и
   направит обратно к `install.sh`, который спрашивает пароль через
   собственный терминал `/dev/tty` — это и есть правильный запасной вариант,
   а не то, что нужно обходить здесь.
