# telegram-mcp-ag

MCP-сервер для чтения Telegram с транскрипцией голосовых сообщений.
Тонкая обёртка над [`chigwell/telegram-mcp`](https://github.com/chigwell/telegram-mcp):
апстрим даёт чтение чатов, мы добавляем расшифровку голосовых, кружков и аудио
через нативный Telegram API.

**Аудитория — не разработчики.** Это определяет всё остальное: установка должна
проходить в одно действие, а сообщения об ошибках — быть понятны человеку,
который не открывал терминал. Подробный план работ — в [PLAN.md](./PLAN.md).

## Стек

```
Python ≥3.10 / Telethon (MTProto) / MCP SDK / setuptools
upstream chigwell/telegram-mcp, закреплён по SHA
```

## Структура

```
src/telegram_mcp_ag/
├── server.py     # MCP-инструменты транскрипции поверх апстрима
└── __init__.py
tests/            # pytest, без сети
examples/         # образцы конфигов клиентов (только плейсхолдеры)
install.sh        # установщик для macOS/Linux
install.ps1       # установщик для Windows
```

## Инварианты — не ломать

1. **Read-only по умолчанию.** `TELEGRAM_EXPOSED_TOOLS=read-only`. Отправка сообщений
   и любые изменяющие действия не включаются без явного решения: сессия Telegram
   равна полному доступу к аккаунту пользователя.
2. **Секреты только в `~/telegram-mcp-ag/config.env`** с правами `600`. Никогда —
   в конфигах клиентов (`.mcp.json`, `.codex/config.toml`), потому что их случайно
   коммитят. В репозиторий не попадает ничего, кроме `*.example`.
3. **Апстрим закреплён по SHA.** Обновлять осознанно, прогнав тесты: `runtime.py`
   апстрима — наша единственная точка интеграции, и она не имеет гарантий совместимости.
4. **Проверять доступ до запроса, а не ловить исключение.** Транскрипция требует
   Telegram Premium либо расходует недельную квоту бесплатного аккаунта. Пользователь
   должен получать внятный текст про остаток квоты, а не ошибку Telegram.
5. **Одна сессия — одна машина.** Telegram отвечает `AUTH_KEY_DUPLICATED` и убивает
   сессию, если одна строка используется с разных IP. Для второй машины — новый вход,
   а не копирование `TELEGRAM_SESSION_STRING`.
6. **Публичный репозиторий.** Никаких личных путей, инфраструктуры, реальных
   `api_id`/`api_hash`/session-строк — ни в коде, ни в примерах, ни в истории git.

## Что транскрибируется

Работает: голосовые сообщения, видео-кружки, аудиофайлы и подкасты.
Не работает: обычные видео, документы с аудио внутри.

## Поддерживаемые клиенты

| Клиент | Как |
| :--- | :--- |
| Claude Code | `claude mcp add -s user` |
| Claude Desktop | `.mcpb`-бандл (двойной клик) или правка `claude_desktop_config.json` |
| Codex CLI | `codex mcp add` → `~/.codex/config.toml` |
| ChatGPT Desktop | тот же `~/.codex/config.toml` |
| ChatGPT web/mobile | **не поддерживается** — там только remote HTTPS MCP, не stdio |

## Разработка

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -e ".[dev]"
.venv/bin/python -m pytest
```

Тесты не ходят в сеть и не требуют Telegram-сессии.
