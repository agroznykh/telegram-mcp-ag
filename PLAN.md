## Ключевые решения (зафиксировано, не переобсуждать)

1. **Секреты не живут в конфигах клиентов.** Единый файл `~/telegram-mcp-ag/config.env`
   с правами `600`. В конфиге клиента — только путь к исполняемому файлу.
   Тогда случайный коммит `.mcp.json` не утечёт, а перевыпуск сессии — правка одного файла.

2. **Одна сессия = одна машина.** `api_id`/`api_hash` переиспользуются свободно,
   `TELEGRAM_SESSION_STRING` — нет: Telegram отвечает `AUTH_KEY_DUPLICATED` и убивает
   сессию. Для ноута и VPS генерируются **разные** сессии с разными
   `TELEGRAM_DEVICE_MODEL`. Установщик на второй машине проводит новый вход,
   а не предлагает скопировать строку.

3. **Premium — не бинарный флаг.** У не-Premium есть недельная бесплатная квота
   (`transcribe_audio_trial_weekly_number`, порядка 2 сообщений) с лимитом длительности.
   Проверяем программно **до** отправки запросов, а не ловим `PremiumAccountRequiredError`.

4. **Что транскрибируется — говорим прямо.** Работает: голосовые, видео-кружки,
   аудиофайлы/подкасты. Не работает: обычные видео, документы с аудио внутри.

5. **Три пути установки** (не альтернативы, а слои под разную аудиторию):
   - `.mcpb`-бандл — Claude Desktop, двойной клик
   - `install.sh` / `install.ps1` — всё остальное, одна строка
   - `pip install git+...` — разработчики
   `pip` вызывается **внутри** установщика, пользователь его не видит.

6. **`uv` — ускоритель, не требование.** Каскад: есть `uv` → есть системный Python ≥3.10
   → предложить поставить `uv` (без sudo, сам скачает Python) → ссылка на python.org.

7. **Клиенты.** Claude Code, Claude Desktop, Codex CLI, ChatGPT Desktop — поддерживаем.
   ChatGPT web/mobile — **не поддерживаем**, там только remote HTTPS MCP.
   Регистрация в `~/.codex/config.toml` покрывает Codex CLI + ChatGPT Desktop + IDE-расширение сразу.

8. **Кроссплатформенность** — macOS, Linux, Windows. Различаются: путь к Python,
   путь к конфигу Claude Desktop, отрисовка QR в терминале.

## Шаг 3. Проверка Premium и доработки сервера — готово ✅

**Модель: Opus 4.8** (тонкая логика вокруг API Telegram) · тот же чат, что шаг 2

1. Новый tool `check_transcription_access` → Premium да/нет, остаток триала,
   дата сброса, максимальная длительность
2. Предварительная проверка в `transcribe_voice_messages`: если не Premium и запрошено
   больше остатка — вернуть понятный ответ, **не отправляя** запросы
3. `PremiumAccountRequiredError` → человеческий текст про Premium и сброс квоты
4. Убрать `from telegram_mcp.runtime import *` — явные импорты, чтобы обновление
   апстрима ломалось громко, а не молча
5. Починить `transcribe_voice_message`: не пробрасывает `request_timeout`
6. Явная ошибка для неподдерживаемых типов медиа (обычное видео, документ)
7. Тесты на новую логику с замоканным клиентом

**Критерий готовности:** на не-Premium аккаунте вызов транскрипции даёт внятный текст
про квоту, а не ошибку Telegram.

## Шаг 4. Установщик `install.sh` (macOS / Linux)

**Модель: Opus 4.8** (много ветвлений и обработки ошибок) · **новый чат**

Запуск: `curl -fsSL https://raw.githubusercontent.com/agroznykh/telegram-mcp-ag/main/install.sh | bash`

1. Определить ОС и архитектуру
2. Каскад поиска Python (см. решение 6), при необходимости предложить установку `uv`
3. Создать `~/telegram-mcp-ag/`, виртуальное окружение
4. `pip install git+https://github.com/agroznykh/telegram-mcp-ag.git@<tag>`
5. Интерактивный опрос: `api_id`, `api_hash` со ссылкой на my.telegram.org/apps
   и объяснением, где их взять
6. Вход в Telegram: QR по умолчанию, фолбэк на телефон + код
7. Записать `config.env` с `chmod 600`
8. Детект установленных клиентов и регистрация:
   - `claude mcp add -s user telegram -- ~/telegram-mcp-ag/.venv/bin/telegram-mcp-ag`
   - `codex mcp add ...` с фолбэком на прямую правку `~/.codex/config.toml`
     (команда помечена экспериментальной, формат может измениться)
   - правка `claude_desktop_config.json` по путям для конкретной ОС
9. Бэкап каждого конфига перед правкой
10. Идемпотентность: повторный запуск обновляет, не плодит дубликаты
11. Финальная самопроверка: подключение к Telegram + `check_transcription_access`
12. Итоговое сообщение: что установлено, куда, как проверить, как удалить
13. `install.sh --uninstall`: снять регистрации, удалить папку, напомнить про отзыв сессии

**Критерий готовности:** на чистой машине человек, не открывавший терминал,
доходит до работающего сервера, введя одну строку и ответив на два вопроса.
