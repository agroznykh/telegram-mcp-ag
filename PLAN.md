## Ключевые решения (зафиксировано, не переобсуждать)

1. **Секреты не живут в конфигах клиентов.** Единый файл `~/telegram-mcp-ag/config.env`
   с правами `600`. В конфиге клиента — только путь к исполняемому файлу.
   Тогда случайный коммит `.mcp.json` не утечёт, а перевыпуск сессии — правка одного файла.

2. **Одна сессия = одна машина.** `api_id`/`api_hash` переиспользуются свободно,
   `TELEGRAM_SESSION_STRING` — нет: Telegram отвечает `AUTH_KEY_DUPLICATED` и убивает
   сессию. Для ноута и VPS генерируются **разные** сессии с разными
   `TELEGRAM_DEVICE_MODEL`. Установщик на второй машине проводит новый вход,
   а не предлагает скопировать строку.

3. **Premium — не бинарный флаг.** У не-Premium есть недельная бесплатная квота
   (`transcribe_audio_trial_weekly_number`, порядка 2 сообщений) с лимитом длительности.
   Проверяем программно **до** отправки запросов, а не ловим `PremiumAccountRequiredError`.

4. **Что транскрибируется — говорим прямо.** Работает: голосовые, видео-кружки,
   аудиофайлы/подкасты. Не работает: обычные видео, документы с аудио внутри.

5. **Три пути установки** (не альтернативы, а слои под разную аудиторию):
   - `.mcpb`-бандл — Claude Desktop, двойной клик
   - `install.sh` / `install.ps1` — всё остальное, одна строка
   - `pip install git+...` — разработчики
   `pip` вызывается **внутри** установщика, пользователь его не видит.

6. **`uv` — ускоритель, не требование.** Каскад: есть `uv` → есть системный Python ≥3.10
   → предложить поставить `uv` (без sudo, сам скачает Python) → ссылка на python.org.

7. **Клиенты.** Claude Code, Claude Desktop, Codex CLI, ChatGPT Desktop — поддерживаем.
   ChatGPT web/mobile — **не поддерживаем**, там только remote HTTPS MCP.
   Регистрация в `~/.codex/config.toml` покрывает Codex CLI + ChatGPT Desktop + IDE-расширение сразу.

8. **Кроссплатформенность** — macOS, Linux, Windows. Различаются: путь к Python,
   путь к конфигу Claude Desktop, отрисовка QR в терминале.

## Шаг 6б. Вход в Telegram прямо в чате (QR) — готово ✅

**Модель: Opus 4.8** · после шага 6

Обнаружено при работе над шагом 6: форма Claude Desktop одноразовая и обратного
канала не имеет, а вход в Telegram интерактивен. `api_id`/`api_hash` формой
собрать можно, а сессию — нет. Апстрим это подтверждает своим же сообщением:
«Interactive phone login is disabled for the MCP server because it runs over stdio».
Поэтому сейчас бандл берёт сессию из `~/telegram-mcp-ag/config.env`, то есть
пользователю всё ещё нужен `install.sh`. Это ломает цель «установка в одно действие»
именно для Claude Desktop.

**Решение — QR, а не телефон+код.** Через чат проходит только одноразовый
login-токен картинкой: ни номера, ни кода, ни пароля 2FA, ни session-строки.
Токен привязан к ключу нашего клиента, а подтверждение происходит на телефоне,
поэтому перехват картинки войти в аккаунт не позволяет. Вариант «телефон + код»
отвергнут: код и, хуже того, пароль 2FA уходили бы в переписку.

1. [x] Режим «только логин». Реализован иначе, чем задумывалось: `runtime`
   импортируется всегда, потому что без него нет ни объекта `mcp`, ни клиента.
   Апстрим строит клиентов на импорте и выходит без сессии, поэтому ему
   подсовывается временная файловая сессия, которую `_install_login_client()`
   тут же меняет на `StringSession()` в памяти — до любого подключения, так что
   ключ на диск не попадает. Инструменты чтения при этом снимаются с регистрации:
   без сессии они дали бы только ошибки авторизации
2. [x] Инструменты `telegram_login_start` / `telegram_login_check` (Telethon
   `qr_login()`), зависимости `qrcode` + `pypng` (pure-Python PNG вместо Pillow)
3. [x] Session-строка в чат не печатает­ся — есть тест
4. [x] Перезапуск не нужен: после входа тот же клиент авторизован на месте,
   инструменты возвращаются в реестр и уходит `notifications/tools/list_changed`.
   Если хост уведомление не принял — просьба переключить расширение
5. [ ] **Осталось на живой машине:** установка двойным кликом → форма → вход
   по QR в чате → чтение чатов

**Критерий готовности:** человек ставит бандл двойным кликом и логинится в чате,
`install.sh` для Claude Desktop больше не нужен.

### Что выяснилось по ходу и изменило решения

- **Картинку в чате нельзя считать доставленной.** Клиенты Anthropic сворачивают
  блок вызова инструмента, и `ImageContent` либо прячется за «развернуть», либо
  не рендерится вовсе (`anthropics/claude-code#53256`, `anthropics/claude-ai-mcp#238`);
  плюс лимит ~1 МБ на контент. Поэтому QR идёт тремя каналами сразу: картинкой,
  файлом PNG, который сервер открывает системным просмотрщиком (сервер работает
  на машине пользователя), и ссылкой `tg://login?token=…` текстом
- **QR не закрывает 2FA.** `qr_login().wait()` бросает `SessionPasswordNeededError`,
  и без облачного пароля вход не завершить (`LonamiWebs/Telethon#3665`). Пароль
  спрашивается **системным окном** (`zenity`/`kdialog`, `osascript`, PowerShell),
  используется один раз и не сохраняется никуда: ни в чат, ни в файл, ни в keychain.
  Вариант «поле в форме бандла» отвергнут — постоянно хранить облачный пароль
  хуже, чем session-строку, которую он и так открывает. Если окна нет
  (headless-Linux) — честный фолбэк на `install.sh`
- **Битая session-строка** больше не роняет сервер трейсбеком: она отбрасывается
  с предупреждением и превращается в обычный сценарий «сессии нет» → вход по QR.
  Это закрывает открытый вопрос 3
