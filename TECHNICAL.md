# Технические детали

Этот документ — для тех, кому интересно, как устроен telegram-mcp-ag внутри.
Для установки и обычного использования он не нужен — см. [README.md](README.md).

## Что это такое

Тонкая обёртка над апстрим-проектом
[`chigwell/telegram-mcp`](https://github.com/chigwell/telegram-mcp): он даёт
чтение чатов через MTProto (нативный протокол Telegram), а этот проект
добавляет расшифровку голосовых, видео-кружков и аудио — тоже через нативный
API Telegram, без сторонних сервисов распознавания речи.

Апстрим закреплён на конкретном коммите (SHA указан в `pyproject.toml`), а не
на плавающей версии: `telegram_mcp.runtime` из апстрима — единственная точка
интеграции, и она не даёт гарантий совместимости между коммитами. Обновление
пина — осознанный шаг с проверкой, что изменилось, а не рутинное обновление
зависимости.

Оба проекта — этот и апстрим — распространяются под лицензией **Apache-2.0**.

## Стек

```
Python ≥3.10 / Telethon (MTProto) / MCP SDK / setuptools
```

## Read-only режим

Сервер поднимается с `TELEGRAM_EXPOSED_TOOLS=read-only` — из всех
MCP-инструментов регистрируются только помеченные `readOnlyHint=True`.
Отправка сообщений и любые изменяющие действия физически недоступны в
конфигурации по умолчанию, а не просто «не упомянуты в подсказках».

## MCP-инструменты

- `check_transcription_access` — статус Premium, остаток бесплатной недельной
  квоты на расшифровку, дата её обновления.
- `transcribe_voice_messages` — расшифровка списка сообщений; сам разбивает
  список на части с учётом лимита времени одного вызова MCP, так что не нужно
  вызывать его по одному сообщению.
- `transcribe_voice_message` — то же самое для одного сообщения.
- `telegram_login_start` / `telegram_login_check` — вход по QR-коду прямо в
  чате, для случаев, когда сессии ещё нет (используется бандлом `.mcpb` для
  Claude Desktop и скиллом `setup-telegram-mcp`).
- Плюс весь набор инструментов чтения из апстрима (чаты, папки, контакты,
  история сообщений) — в режиме `read-only`.

## Как хранятся секреты

Все секреты (`api_id`, `api_hash`, строка Telegram-сессии) живут в одном
файле — `~/telegram-mcp-ag/config.env`, который создаётся с правами `600`
(чтение и запись только владельцу; на Windows аналогичное ограничение
обеспечивает ACL файловой системы). В конфигах MCP-клиентов
(`.mcp.json`, `~/.codex/config.toml`, `claude_desktop_config.json`) остаётся
только путь к исполняемому файлу — сами секреты туда никогда не попадают,
потому что такие файлы гораздо чаще случайно коммитят в git или показывают
на скриншоте.

Строка сессии Telegram равносильна полному доступу к аккаунту. Подробный
разбор угроз и что делать при подозрении на утечку — в [SECURITY.md](SECURITY.md).

### Вход по QR в чате

Через переписку с ассистентом проходит только одноразовый login-токен
картинкой — не номер телефона, не код, не пароль двухфакторной аутентификации
и не строка сессии. Если у аккаунта включён облачный пароль (2FA), его
запрашивает системное окно ОС (`zenity`/`kdialog` на Linux, `osascript` на
macOS, защищённый ввод PowerShell на Windows) — используется один раз и не
сохраняется никуда: ни в чат, ни в файл, ни в системное хранилище паролей.

## Почему не поддерживаются ChatGPT web и мобильное приложение

Этот сервер — локальный процесс, который общается с клиентом по протоколу
stdio (стандартный ввод/вывод), а не по сети. ChatGPT web и мобильное
приложение умеют подключать только **удалённые** MCP-серверы по HTTPS —
то есть потребовался бы отдельный сервер, доступный из интернета, которому
нужно было бы доверить сессию Telegram. Вместо этого сессия хранится только
локально, на компьютере пользователя, поэтому эти два клиента в принципе не
могут быть поддержаны без изменения всей модели угроз проекта.

## Одна сессия — одна машина

`api_id`/`api_hash` можно свободно переиспользовать между машинами — это не
секрет уровня сессии, они привязаны к Telegram-приложению, а не к конкретному
входу. А вот `TELEGRAM_SESSION_STRING` — нет: если Telegram видит одну и ту же
строку сессии с разных IP, он отвечает `AUTH_KEY_DUPLICATED` и убивает обе
сессии. Поэтому для второй машины устанавливается отдельный вход
(`install.sh --relogin` / `install.ps1 -Relogin`), а не копия строки.

## Структура репозитория

```
src/telegram_mcp_ag/
├── server.py     # MCP-инструменты транскрипции поверх апстрима, выбор режима
├── login.py      # вход по QR прямо в чате, когда сессии ещё нет
├── config.py     # креды из окружения и ~/telegram-mcp-ag/config.env
└── __init__.py
tests/            # pytest, без сети
examples/         # образцы конфигов клиентов (только плейсхолдеры)
mcpb/             # бандл .mcpb для Claude Desktop (manifest + сборщик)
install.sh        # установщик для macOS/Linux
install.ps1       # установщик для Windows
```
